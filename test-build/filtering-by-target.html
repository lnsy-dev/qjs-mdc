Filtering by Target

The --target flag lets you compile only the markdown files intended for a specific destination, using the target field in YAML front matter.1 See Setting Up a Markdown File for a full list of supported front matter fields.

Front Matter

Add a target field to any markdown file:

---
publish: true
title: My Post
date: 2026-03-04T00:00:00
target: tiny-ag.blog
---

Usage

./mdc notebook --output docs --target tiny-ag.blog

Only files where target: tiny-ag.blog will be compiled. Files with a different target value or no target field are skipped.2

Without the Flag

Running without --target compiles all publishable files regardless of their target field:

./mdc notebook --output docs

This is the default behavior and is backwards-compatible — existing files without a target field continue to work as before.

  1. This is useful when a single notebook directory contains content for multiple sites or audiences. Each `./mdc` invocation can produce a different output set from the same source. ↩
  2. A file with no `target` field is also skipped when `--target` is active. Only an exact string match on the `target` field value causes a file to be included. ↩
